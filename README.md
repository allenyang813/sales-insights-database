# Sales Insights Project

## Overview
This project involved designing a relational database for a furniture business and writing SQL queries to extract meaningful business information. The assignment included building multiple related tables, enforcing key constraints, inserting sample data, and developing analytical queries.

## What I Did
- Designed a normalised relational schema including Customers, Staff, Orders, OrderItems, Payments, and Customer Types.
- Implemented tables with primary keys, foreign keys, and composite keys to enforce referential integrity.
- Inserted sample data for customers, staff members, sales orders, items, and payments.
- Wrote SQL queries to:
  - Retrieve customers with specific name patterns.
  - Determine each customer's earliest order date.
  - Calculate total order amounts using joins and aggregates.
  - Compute payments received per order.
  - Count orders per staff member across multiple years using CROSS JOIN.
  - Calculate outstanding balances based on payments received.

## Technologies Used
- SQL (MySQL)
- Relational database modelling
- JOINs, GROUP BY, aggregate functions, composite keys
